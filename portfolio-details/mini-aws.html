AWS Infrastructure as Code | Génération automatique via JSON

Description du projet

Principe

Ce projet consiste à concevoir un outil en Python capable de générer automatiquement une infrastructure réseau complète sur AWS à partir d’un fichier de configuration JSON.

L’idée centrale est de permettre à un utilisateur de décrire une architecture cloud de manière déclarative, via une structure JSON claire et hiérarchisée, puis de déployer cette infrastructure sans intervention manuelle.

Le fichier JSON définit les composants essentiels d’un environnement AWS : VPC, sous-réseaux publics et privés, passerelles Internet et NAT, tables de routage, groupes de sécurité, NACLs et instances EC2.

Le script Python lit cette spécification et crée les ressources en respectant leurs dépendances logiques, en s’appuyant sur la bibliothèque boto3, SDK officiel AWS pour Python.

But

L’objectif du projet est d’automatiser le déploiement d’infrastructures cloud afin de réduire les erreurs humaines et améliorer la reproductibilité.

Il s’agit également de :

  • Concevoir un format JSON structuré et cohérent
  • Gérer les dépendances entre ressources AWS
  • Implémenter un déploiement automatisé fiable
  • Faciliter la création et la suppression d’environnements

Le projet s’inscrit dans une approche Infrastructure as Code (IaC), où l’infrastructure est définie et versionnée comme du code.

Finalité

Le projet aboutit à un outil fonctionnel permettant de déployer une infrastructure AWS complète à partir d’un simple fichier JSON.

Il constitue une première approche concrète des pratiques DevOps, en particulier l’automatisation du provisioning d’infrastructure.
