Détection d'intrusions | IA pour injections SQL sur Raspberry Pi

Description du projet

Contexte

Ce projet porte sur la détection d’intrusions dans un contexte de cybersécurité, plus précisément sur l’identification des attaques par injection SQL ciblant des applications web.

L’objectif est de développer une solution basée sur l’intelligence artificielle capable de détecter en temps réel des tentatives d’exploitation SQL, afin de renforcer la sécurité des systèmes d’information.

Principe

La solution repose sur l’exploitation d’un dataset nommé sql-injection-dataset.csv, contenant des exemples de requêtes légitimes et malveillantes.

Un modèle de machine learning est entraîné afin de classifier les requêtes et détecter les comportements suspects. Le choix du modèle privilégie un compromis entre performance et légèreté afin de permettre un déploiement embarqué.

Le système est conçu pour fonctionner sur un Raspberry Pi, utilisé comme dispositif de protection réseau (“security shield”) intégré à l’infrastructure existante.

Méthodologie

Le développement s’articule autour des étapes suivantes :

  • Analyse et préparation du dataset
  • Choix et entraînement d’un modèle d’IA adapté
  • Optimisation pour un déploiement sur matériel embarqué
  • Tests sur environnement réel ou simulé

L’environnement Python (notamment via notebook) est utilisé pour accélérer l’expérimentation et l’itération.

Contraintes

Le projet impose des contraintes fortes :

  • Modèle léger pour exécution sur Raspberry Pi
  • Détection en quasi temps réel
  • Déploiement rapide dans un contexte de démonstration

Finalité

Le projet aboutit à un prototype fonctionnel démontrant la faisabilité d’un système de détection d’intrusions basé sur l’IA, capable d’être intégré dans une infrastructure réseau existante.

Il s’inscrit dans une démarche de cybersécurité proactive visant à réduire les risques liés aux attaques par injection SQL et à améliorer la résilience des applications web.
